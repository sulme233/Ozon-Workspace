Ozon Performance API 文档 (2.0)
在文件中描述了Ozon Performance API — 用于与广告管理面板进行交换的接口方法 系统卖家和Ozon之间的信息。

关于使用API的工作问题，请通过个人中心联系客服。

介绍
要使用Ozon广告活动报告API，请在您的个人中心获取 client_id 和 client_secret：

在个人中心进入页面 设置 → API-密钥.
如果需要，请创建服务账号：点击 创建新账号。
点击服务账号名称。
点击 添加新密钥。
用于使用API：

使用 client_id 和 client_secret 在授权服务中获取授权令牌。
使用授权令牌执行请求。如果令牌已过期，请获取新的令牌。
如何获取API-密钥
要获取授权令牌，请发送请求：

POST /api/client/token HTTP/1.1
Host: https://api-performance.ozon.ru
Content-Type: application/json
Accept: application/json

{
  "client_id":"XYZ@advertising.performance.ozon.ru", 
  "client_secret":"b1u5XXDQW3wEqQ7dG...ancMyuhQtMNBI", 
  "grant_type":"client_credentials"
}
服务会返回带有有效期秒数的令牌：

{
  "access_token": "eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9.WyJFTWhVVU...qTz2XXZBv41h4",
  "expires_in": 1800,
  "token_type": "Bearer"
}
请求限制
每天的请求总限额为100,000。

在报告生成的开始检查上传统计数据的限制。

限额适用于来自统计部分的报告。

一条广告活动 = 一条导出。

如果请求中有多个广告活动，这将被视为多个导出。

每天的导出数量限制按以下公式计算：活跃广告数量 × 240 — 但不超过以下表格中的值。

下载限制：

限制	限额的含义
导出文件的天数限制	62
报告中的广告数量限制	10
账户同时导出数量的限制	1
每天账户的下载数量限制	2000
组织同时下发数量的限制	5
在组织范围内的24小时内的下载数量限制	2000
